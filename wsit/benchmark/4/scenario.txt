Username For Certificate, Sign then Encrypt
-------------------------------------------
[WSS1.1] SOAP 1.2 message, Request is signed using DKT1(K) (symmetric key derived from K, 
represented using derived key), then  encrypted using a DKT2(K), K is ephemeral key protected for 
Server's Cert. UsernameToken is included in the request signed using DKT1(K) and encrypted using 
DKT2(K). Response is signed using DKT3(K), encrypted using DKT4(K)
