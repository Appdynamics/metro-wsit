Enabling/Disabling WS-Addressing Member Submission Version in Milestone 1

1.0 Introduction

The WS-Addressing implementation in milestone 1 uses W3C Candidate Recommendation version as the default. This document describes how to override the default implementation to enable WS-Addressing Member Submission version.

2.0 Enabling WS-Addressing Member Submission Version

This step is required if Member Submission version of WS-Addressing needs to be used instead of W3C Candidate Recommendation version.

2.1 Client-side

  1. Copy this jar in your classpath. For example, this may be copied in lib/runtime directory.

2.2 Server-side

  1. If you are using Glassfish
    1. Copy this jar in AS_HOME/lib directory where AS_HOME is the directory where Glassfish is installed.
  2. If you are using Tomcat
    1. Copy this jar in TOMCAT_HOME/shared/lib directory where TOMCAT_HOME is the directory where Tomcat is installed.

3.0 Disabling WS-Addressing Member Submission Version

This step is required if Member Submission version of WS-Addressing is enabled before and now W3C Candidate Recommendation version needs to be used again.

3.1 Client-side

  1. Remove the jar copied in 2.1 from your classpath.

3.2 Server-side

  1. Remove the jar copied in 2.2 from the directory.

4.0 Interop Tests

The following interop tests are known to use Member Submission version of WS-Addressing and will need to make this change to work:

  1. WS-Security 1.0
  2. WS-Secure Conversation
  3. SOAP 1.2 + MTOM
